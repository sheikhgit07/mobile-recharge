# Nexitel.org Laravel CMS (Full Front + Backend)

This repo contains a ready-to-apply scaffold to create a full Laravel app for a dynamic, CMS-driven site similar to `nexitel.org`.

It includes:
- Dynamic pages/sections stored in DB (JSON-driven components)
- Frontend rendered with Blade and Tailwind (via CDN for zero-build startup)
- Admin panel via Filament (generated by setup scripts)
- SEO (meta fields), menus, media, and settings

## Quick Start

Pick ONE of the two options below.

### Option A: With Docker (recommended)

Requirements: Docker + Docker Compose

1) Bootstrap a fresh Laravel app with Sail and apply this scaffold

```bash
bash scripts/setup_with_docker.sh
```

2) When finished, open the app:
- App URL: http://localhost
- Admin URL: http://localhost/admin (Filament)
- Admin user: seeded as `admin@example.com` with password `password` (change ASAP)

### Option B: Without Docker

Requirements: PHP 8.2+, Composer, Node 18+ (optional), MySQL or SQLite

1) Install Laravel

```bash
composer create-project laravel/laravel app
cd app
```

2) Apply the scaffold

```bash
bash ../scripts/setup_without_docker.sh
```

3) Configure `.env`
- Set `APP_URL=http://localhost`
- Configure database (MySQL recommended) or fallback to SQLite

4) Run migrations and seeders

```bash
php artisan migrate --force
php artisan db:seed --class=Database\Seeders\CmsSeeder --force
```

5) Serve the app

```bash
php artisan serve
```

Open:
- App: http://127.0.0.1:8000
- Admin: http://127.0.0.1:8000/admin

Admin credentials (seeded):
- Email: `admin@example.com`
- Password: `password`

## What’s Inside

- scripts/
  - setup_with_docker.sh: Fully automated Docker-based setup using Laravel Sail
  - setup_without_docker.sh: Composer-based setup and scaffold apply
  - apply_scaffold.sh: Copies files from `scaffold/` into the Laravel app and installs Filament
- scaffold/
  - app/, database/, routes/, resources/ …
  - Models: Page, Section, Menu, MenuItem, Media, Setting
  - Controllers: PageController (frontend), basic API
  - Migrations + Seeders: CMS schema + example homepage
  - Views: Layout + componentized sections (hero, features, pricing, testimonials, contact, footer)
  - Filament admin resources for all content types

## Notes
- Tailwind is loaded via CDN in the base layout to avoid a build step. You can switch to a full Node build later.
- Filament requires storage link and some publish steps handled by `apply_scaffold.sh`.
- For production, set up a proper DB (MySQL/PG), queue, cache, and assets build.
